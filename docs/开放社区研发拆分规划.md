# 开放社区研发拆分规划

## 1. 文档说明

- 需求来源：`docs/开放社区.xlsx`
- 目标：将原始功能清单拆成可执行的研发任务包，按 `MVP / V1 / V2` 三阶段组织，直接用于排期、分工和后续接口设计。
- 适用范围：`web` 前台与后台、`server` API 服务、数据存储、对象存储、搜索、联调与验收。
- 默认约束：
  - `web` 使用单一前端工程，同时承载门户前台和后台管理端，通过不同路由分区实现。
  - `server` 使用单一 Gin 服务，按模块提供 REST API。
  - 上传文件统一走对象存储抽象，模型与数据集下载不直接暴露物理存储路径。
  - 后台管理端纳入本次规划范围，审核、推荐、公告、分类、认证、权限必须同步规划。
  - 门户与社区类页面的视觉与信息架构参考 `https://forum.openloong.org.cn/`，重点借鉴其浅色背景、顶部导航、内容分区、卡片化模块和较高信息密度的社区门户布局，不做逐像素复制。

## 2. 拆分原则

- 高优先级需求优先进入 `MVP`，但可按可上线闭环拆分为“核心能力先上、增强能力后补”。
- 中优先级需求优先进入 `V1`，低优先级需求默认进入 `V2`。
- 所有模块统一拆成五类任务：
  - `[FE]` 前端页面与交互
  - `[BE]` 后端接口与业务编排
  - `[DB]` 数据对象与存储设计
  - `[ADMIN]` 后台运营与审核能力
  - `[QA]` 联调、验收与回归场景
- 任务拆到接口级，但不在本文中继续细化到字段定义和具体表结构字段名；字段设计在后续接口设计文档中完成。

## 4. 公共基础能力

这些能力不是单独的业务模块，但属于所有阶段都要考虑的共用底座。

### 4.1 账号、权限与审核状态机

- `[BE]` 统一用户、角色、权限、资源所有者、审核状态流转能力。
- `[DB]` 建立用户、角色、权限、审核记录、资源归属关系。
- `[ADMIN]` 提供基础审核操作，包括通过、驳回、下架、推荐、屏蔽。
- `[QA]` 覆盖普通用户、开发者、运营管理员的权限边界。

### 4.2 文件与对象存储

- `[BE]` 统一上传凭证、文件元信息、下载授权、下载日志。
- `[DB]` 建立文件资源表、对象路径映射、下载记录。
- `[QA]` 覆盖大文件上传、断点重试、无权限下载、下载记录写入。

### 4.3 标签、分类与检索

- `[BE]` 抽象标签、分类、场景、机器人类型等通用字典。
- `[DB]` 建立通用标签关系表、分类表、热度统计表。
- `[ADMIN]` 后台维护标签、分类、推荐位、公告。

### 4.4 评论、评分、通知

- `[BE]` 统一评论、评分、回复、系统通知能力，供模型、数据集、模板、技能复用。
- `[DB]` 建立评论、评分、通知、已读状态、收藏关系。
- `[QA]` 覆盖跨资源评论、重复评分、通知已读和未读计数。

## 5. MVP 研发任务包

目标：完成“用户可注册登录、资源可上传审核、内容可浏览搜索、文件可下载、后台可运营”的最小上线闭环。

### 5.1 门户网站

关联需求：`5010101`、`5010102`、`5010103`

- `[FE]` 实现首页、全局搜索页、登录注册页、个人中心入口。
- `[FE]` 首页模块固定为：平台介绍、推荐资源、最新公告、应用场景。
- `[FE]` 搜索结果页先支持按资源类型、标签、时间、热度、适用机器人筛选。
- `[FE]` 门户首页与后续社区页统一参考 OpenLoong 论坛的门户布局语言，优先采用“顶部导航 + 导读区 + 内容分区卡片”的结构。
- `[BE]` 提供 `GET /api/v1/portal/home`，返回首页聚合数据。
- `[BE]` 提供 `GET /api/v1/search`，支持跨模型、数据集、文档、任务模板的统一检索。
- `[BE]` 提供 `POST /api/v1/auth/register`、`POST /api/v1/auth/login`、`POST /api/v1/auth/logout`、`GET /api/v1/users/me`。
- `[DB]` 建立公告、推荐位、搜索热度统计、登录会话相关对象。
- `[ADMIN]` 后台提供公告发布、推荐位维护、首页模块开关。
- `[QA]` 验收首页聚合、搜索筛选排序、登录态切换和个人中心入口跳转。

### 5.2 模型仓库

关联需求：`5010201`、`5010202`、`5010203`

- `[FE]` 实现模型列表页、模型详情页、模型上传页、我的模型页。
- `[FE]` 列表页支持卡片和列表视图切换，详情页展示介绍、版本、依赖、许可证、下载统计。
- `[BE]` 提供模型接口族：
  - `GET /api/v1/models`
  - `GET /api/v1/models/{id}`
  - `POST /api/v1/models`
  - `PUT /api/v1/models/{id}`
  - `POST /api/v1/models/{id}/versions`
  - `POST /api/v1/models/{id}/submit`
  - `POST /api/v1/models/{id}/download`
  - `GET /api/v1/users/me/models`
- `[BE]` 模型下载必须写入下载记录、资源热度和用户下载历史。
- `[DB]` 建立模型、模型版本、模型文件、依赖声明、许可证、下载记录、审核记录。
- `[ADMIN]` 后台提供模型审核、上下架、推荐、标签维护。
- `[QA]` 验收上传、版本新增、审核后可见、下载授权、历史记录和无权限下载拦截。

### 5.3 数据集仓库

关联需求：`5010301`、`5010302`、`5010303`

- `[FE]` 实现数据集列表页、详情页、上传页、样本预览区、我的数据集页。
- `[FE]` 样本预览至少支持图片和文本，视频和点云先预留组件占位与接口。
- `[BE]` 提供数据集接口族：
  - `GET /api/v1/datasets`
  - `GET /api/v1/datasets/{id}`
  - `POST /api/v1/datasets`
  - `PUT /api/v1/datasets/{id}`
  - `POST /api/v1/datasets/{id}/versions`
  - `GET /api/v1/datasets/{id}/samples`
  - `POST /api/v1/datasets/{id}/download`
  - `POST /api/v1/datasets/{id}/agreements/confirm`
- `[BE]` 分批次下载在 MVP 先实现为打包任务和分片链接，不做复杂下载客户端。
- `[DB]` 建立数据集、数据集版本、样本元信息、协议确认记录、下载记录、隐私权限设置。
- `[ADMIN]` 后台提供数据集审核、分类、协议模板、权限级别维护。
- `[QA]` 验收上传、样本预览、协议确认、分批下载、审核后展示、下载日志。

### 5.4 任务模板与具身应用

关联需求：`5010401`、`5010403`

- `[FE]` 实现任务模板列表页、模板详情页、具身应用案例列表页、案例详情页。
- `[FE]` 模板详情展示适用场景、使用说明、依赖资源、操作步骤。
- `[BE]` 提供接口：
  - `GET /api/v1/task-templates`
  - `GET /api/v1/task-templates/{id}`
  - `GET /api/v1/application-cases`
  - `GET /api/v1/application-cases/{id}`
- `[DB]` 建立任务模板、模板分类、应用案例、应用场景标签。
- `[ADMIN]` 后台提供模板维护、案例维护、分类维护。
- `[QA]` 验收模板浏览、场景过滤、案例详情和部署指南展示。

### 5.5 文档中心

关联需求：`5010501`、`5010502`

- `[FE]` 实现平台文档页、技术文档页、FAQ 页、视频教程目录页。
- `[FE]` 文档中心统一使用左侧目录 + 右侧正文布局，支持搜索目录与面包屑。
- `[BE]` 提供接口：
  - `GET /api/v1/docs/categories`
  - `GET /api/v1/docs`
  - `GET /api/v1/docs/{id}`
  - `GET /api/v1/faqs`
- `[DB]` 建立文档分类、文档正文、FAQ、视频链接资源对象。
- `[ADMIN]` 后台提供文档分类、内容发布、FAQ 管理。
- `[QA]` 验收目录展示、正文渲染、FAQ 检索、视频链接跳转。

### 5.6 个人中心与基础通知

关联需求：`5010601`

- `[FE]` 实现个人资料页、我的模型、我的数据集、我的任务、收藏、下载记录、通知中心。
- `[BE]` 提供接口：
  - `GET /api/v1/users/me/profile`
  - `PUT /api/v1/users/me/profile`
  - `GET /api/v1/users/me/favorites`
  - `GET /api/v1/users/me/downloads`
  - `GET /api/v1/users/me/notifications`
  - `POST /api/v1/users/me/notifications/read`
- `[DB]` 建立用户资料扩展、收藏、通知、通知已读状态、贡献基础统计。
- `[ADMIN]` 后台可查看用户贡献和资源归属。
- `[QA]` 验收个人资料更新、收藏记录、下载历史、审核通知与系统通知展示。

## 6. V1 研发任务包

目标：补齐互动、反馈、评价、共享和增强检索，让平台从“资源平台”升级为“社区平台”。

### 6.1 模型评测与对比

关联需求：`5010202`、`5010204`

- `[FE]` 新增模型对比页、评测结果页、反馈表单。
- `[BE]` 新增接口：
  - `POST /api/v1/models/{id}/reviews`
  - `POST /api/v1/models/{id}/ratings`
  - `POST /api/v1/model-evaluations`
  - `GET /api/v1/model-evaluations/{id}`
- `[DB]` 建立评测任务、评测结果、评分、反馈记录。
- `[ADMIN]` 提供评测结果查看、异常反馈处理。

### 6.2 数据集统计增强

关联需求：`5010302`

- `[FE]` 增加数据统计图表、下载趋势、样本结构概览。
- `[BE]` 提供 `GET /api/v1/datasets/{id}/stats`。
- `[DB]` 建立数据集统计快照和下载趋势聚合。
- `[QA]` 验收统计口径与下载日志一致。

### 6.3 技能分享与模板互动

关联需求：`5010401`、`5010402`

- `[FE]` 实现技能发布页、技能详情页、Fork 关系展示、模板评分评论区。
- `[BE]` 提供接口：
  - `GET /api/v1/skills`
  - `GET /api/v1/skills/{id}`
  - `POST /api/v1/skills`
  - `POST /api/v1/skills/{id}/fork`
  - `POST /api/v1/task-templates/{id}/ratings`
  - `POST /api/v1/task-templates/{id}/comments`
- `[DB]` 建立技能、技能版本、Fork 关系、模板评分评论。
- `[ADMIN]` 提供技能审核、违规下架。

### 6.4 互动社区

关联需求：`5010602`

- `[FE]` 实现讨论区、评论回复、关注/粉丝页。
- `[FE]` 讨论区列表、侧边栏和推荐区延续门户页视觉语言，保持社区型信息密度和卡片分区。
- `[BE]` 提供接口：
  - `GET /api/v1/discussions`
  - `POST /api/v1/discussions`
  - `POST /api/v1/comments`
  - `POST /api/v1/users/{id}/follow`
  - `GET /api/v1/users/me/follows`
- `[DB]` 建立讨论帖子、评论回复、关注关系、粉丝关系。
- `[ADMIN]` 提供帖子审核、评论屏蔽、用户禁言。
- `[QA]` 验收跨资源评论、关注取消关注、讨论内容审核。

### 6.5 搜索与推荐增强

- `[FE]` 搜索页增加热门词、关联推荐、最近搜索。

## 8. 统一接口族

- 认证与用户：`/auth`、`/users/me`
- 门户与搜索：`/portal`、`/search`
- 模型：`/models`
- 数据集：`/datasets`
- 任务模板：`/task-templates`
- 技能：`/skills`
- 应用案例：`/application-cases`
- 文档：`/docs`、`/faqs`
- Wiki：`/wiki`
- 互动：`/comments`、`/ratings`、`/discussions`、`/follows`
- 通知与消息：`/notifications`、`/messages`
- 认证：`/developer-verifications`、`/enterprise-verifications`
- 激励：`/rewards`、`/rankings`
- 后台：`/admin/*`

## 9. 核心数据对象

后续建模默认围绕以下核心对象展开：

- 用户域：用户、角色、权限、会话、用户资料、认证申请、企业认证、通知、收藏、关注关系
- 资源域：模型、模型版本、模型文件、模型评测、数据集、数据集版本、样本元信息、协议确认
- 任务域：任务模板、技能、技能版本、Fork 关系、应用案例
- 内容域：公告、推荐位、文档、FAQ、Wiki 词条、Wiki 修订、讨论帖、评论、评分
- 运营域：审核记录、标签、分类、热门词、搜索记录、下载记录、热度统计
- 激励域：积分账本、兑换记录、贡献排行快照

## 10. 分阶段验收标准

### MVP 验收

- 用户可以注册登录并进入个人中心。
- 模型与数据集可以上传、提交审核、审核通过后展示。
- 首页能展示推荐内容和公告。
- 搜索可以跨资源检索并按基础条件过滤。
- 文档中心可以对外提供平台和技术文档。
- 后台能完成资源审核、推荐配置、公告发布和分类维护。

### V1 验收

- 用户可以对模型、模板、技能进行评分评论。
- 技能分享和 Fork 链路可用。
- 讨论区与关注系统上线，社区互动闭环形成。
- 模型评测和数据统计增强能力可用。

### V2 验收

- Wiki 共编、认证、私信、协作空间、积分体系全部闭环。
- 开发者和企业认证可审核、可展示、可授予权益。
- 积分规则和兑换流程稳定运行。

## 11. 建议实施顺序

为减少阻塞，建议按以下顺序推进：

1. 账号权限、对象存储、标签分类、后台基础框架
2. 门户首页、搜索、个人中心基础版
3. 模型仓库 MVP
4. 数据集仓库 MVP
5. 任务模板与具身应用 MVP
6. 文档中心 MVP
7. 互动、技能分享、模型评测、数据统计增强
8. Wiki、认证、私信协作、积分体系
