# 功能完善优化建议

## 1. 文档目的

- 依据 `docs/开放社区研发拆分规划.md` 与 `docs/迭代文档-2026-04-17.md`，整理当前版本后续仍需完善和优化的能力项。
- 目标不是重复罗列已完成功能，而是给出后续可直接进入排期的缺口清单、优先级和建议实施顺序。
- 范围覆盖前台、后台、后端服务、存储与治理能力。

## 2. 当前阶段判断

- 当前项目已经完成 `MVP` 主闭环，并提前实现了部分 `V1 / V2` 能力。
- 现阶段最需要补的不是单一页面，而是“治理、权限、存储、运营配置”这几类底座能力。
- 若继续只补页面功能而不补底座，后续会在权限边界、审核追踪、文件授权、运营配置和数据治理上反复返工。

## 3. 已完成基础

- 门户首页、搜索、登录注册、个人中心基础入口已经可用。
- 模型仓库、数据集仓库、模板、案例、文档中心、FAQ、视频教程已形成基础浏览与管理闭环。
- 数据集样本预览、协议确认、分批下载、下载审批第一版已上线。
- 社区互动、技能分享、Wiki、认证、私信、协作空间、积分体系已有第一版实现。
- 后台已具备资源审核、内容管理、推荐位、公告、协议模板、权限级别、操作日志等基础能力。

## 4. 仍需完善的核心功能

### 4.1 权限体系与审核流细化

优先级：`P0`

现状：

- 当前主要还是“普通用户 / 管理员”两档权限。
- 后台已能做基础审核和内容治理，但还没有更细粒度的角色、权限点和审批流。

本轮已落地：

- 新增角色枚举第一版：
  - 普通用户 `user`
  - 开发者 `developer`
  - 内容运营 `operator`
  - 审核员 `reviewer`
  - 管理员 `admin`
- 新增超级管理员角色：
  - 超级管理员 `super_admin`
- 后端后台路由已按职责拆分权限：
  - `operator` 可进入门户运营与内容管理
  - `reviewer` 可进入审核、访问审批与认证审核
  - `admin` 仍保留全量后台权限
- 开发者认证审核通过后，用户角色会自动从 `user` 升级到 `developer`。
- 前端路由守卫和顶部后台入口已支持按角色跳转与拦截，不再只识别 `admin`。
- 已补回归测试，验证 `operator / reviewer` 的访问边界。
- 后端已从“纯角色路由”升级到“角色 + 权限码矩阵”：
  - 新增后台权限码体系
  - 新增 `PermissionRequired(...)` 中间件
  - 后台路由已按权限码拆分，不再只靠角色名硬编码
- 新增资源级权限矩阵第一版，覆盖：
  - 门户运营
  - 公告管理
  - 搜索运营词
  - 推荐位
  - 模板 / 案例 / 文档 / FAQ / 视频 / 协议 / 权限级别 / 筛选字典 / 模型推荐标签
  - 资源审核 / 数据集访问审批 / 认证审核
  - 社区内容治理 / 私信治理 / 协作空间治理
  - Wiki 管理
  - 积分权益管理
  - 操作日志与后台总览
- 登录态和 `me` 接口已返回当前用户权限列表，前端鉴权不再只能根据角色名判断。
- 前端路由已支持 `requiredPermissions`，后台页面可按权限码拦截。
- 顶部后台入口和后台页面按钮已开始按权限码控制，而不是只看角色。
- 已为高风险操作补二次确认第一版，覆盖：
  - 门户运营页删除场景页 / 推荐位 / 搜索运营词 / 首页亮点
  - 内容管理页删除与批量删除
  - 社区治理页下架 / 删除 / 封禁 / 解封 / 移除成员
- 已补回归测试，验证：
  - `operator / reviewer` 的权限边界
  - `super_admin` 可访问管理员专属能力
  - `me` 接口会返回权限列表

建议补齐：

- 建立更细角色模型：
  - 超级管理员
- 建立资源级权限矩阵：
  - 模型
  - 数据集
  - 文档
  - Wiki
  - 讨论
  - 评论
  - 私信
  - 协作空间
- 审核动作从“通过 / 驳回”扩展到：
  - 下架
  - 屏蔽
  - 推荐
  - 恢复
  - 禁言
  - 封禁
- 增加高风险操作审批或二次确认：
  - 删除内容
  - 批量下架
  - 批量状态变更
  - 权益配置调整
- 补按钮级权限控制，确保前端操作入口与后端权限边界一致。
- 明确审核记录和责任人追踪字段，保证后台审计闭环。

交付建议：

- 先补角色与权限模型。
- 再补后台鉴权中间件和按钮级控制。
- 最后补审批流和审计视图。

### 4.2 文件存储与下载授权升级

优先级：`P0`

现状：

- 当前仍以本地文件存储为主。
- 文件路径仍通过静态目录直接暴露。

风险：

- 不利于后续接入对象存储。
- 不利于做受控下载、签名链接、限时授权和审计。
- 不利于大文件上传、断点续传和多环境部署。

本轮已落地：

- 后端新增对象存储抽象接口 `ObjectStorage`，当前默认由本地存储实现，已为后续接入 MinIO / S3 兼容存储预留统一能力边界。
- 新增统一文件元信息模型：
  - 文件对象 ID
  - 对象路径 `object_key`
  - 原始文件名
  - MIME 类型
  - 文件大小
  - 存储驱动
  - 上传人
  - 存储作用域
  - 创建 / 更新时间
- 新增授权下载链路：
  - `GET /api/v1/files/download/:token`
  - 模型版本文件、数据集版本文件、样本预览文件、分批下载分片、分包说明文件都已改为短时效签名链接访问
- 前端已不再依赖直接暴露的 `/storage/...` 静态路径作为主要下载入口。
- 数据集样本预览、版本文件下载、分包下载任务已统一切换到授权链接。
- 已补回归测试，覆盖：
  - 数据集样本返回授权下载链接
  - 分批下载任务返回授权链接
  - 原有下载审批与下载任务链路保持可用

建议补齐：

- 抽象对象存储接口，兼容：
  - 本地存储
  - MinIO
  - S3 兼容存储
- 建立统一文件元信息对象：
  - 文件 ID
  - 资源归属
  - 对象路径
  - 原始文件名
  - MIME 类型
  - 文件大小
  - 上传人
  - 上传时间
- 下载改为授权式访问：
  - 生成签名下载链接
  - 受限资源单次授权
  - 下载时效控制
- 为大文件补齐：
  - 分片上传
  - 断点重试
  - 上传失败恢复

交付建议：

- 这一项建议在更复杂的数据集权限矩阵之前完成。

本轮已落地：

- 编辑后资源状态会回退为草稿，并支持再次提交审核。
- 模型详情页与数据集详情页已支持历史版本对比。
- 模型详情页与数据集详情页已直接展示最新驳回原因，不再只能在个人中心查看。
- 模型编辑页已升级为字典化表单：
  - 标签改为可搜索、可扩展的多选标签输入
  - 机器人类型改为字典驱动下拉
  - 许可证改为常见许可证下拉

  - 权限级别继续使用后台配置字典
  - 协议模板可直接选择并回填

- 增加模型编辑能力：
  - 编辑基础信息
  - 编辑标签
  - 编辑适用机器人
  - 编辑依赖
  - 编辑许可证
- 增加数据集编辑能力：
  - 编辑元数据
  - 编辑权限级别
  - 编辑协议文本
  - 编辑样本说明
- 完善“我的模型 / 我的数据集”管理页：
  - 草稿
  - 待审核
  - 已发布
  - 已驳回
  - 重新提交

### 4.5 数据集下载审批升级

优先级：`P1`

现状：

- 当前数据集下载审批是第一版闭环。
- 具备申请、审核、通过后下载与生成分批任务的主链路。

本轮已落地：

- 后台审批列表已补充筛选能力：
  - 按申请状态筛选
  - 按权限级别筛选
  - 按资源负责人筛选
  - 按关键词搜索数据集 / 申请人 / 负责人 / 申请理由
- 审批列表返回并展示更多上下文：
  - 数据集名称
  - 权限级别
  - 数据集负责人
  - 申请人
- 这样后台审核时不需要再根据 `dataset_id` 反查资源详情。
- 后台审批页已支持批量通过 / 批量驳回。
- 审批接口已增加“仅允许处理 pending 申请”的冲突保护，避免重复审批覆盖结果。
- 后台审批页已支持审核备注模板，单条审核和批量审核都可复用模板后再编辑说明。
- `restricted` 数据集已支持两级审批：
  - 第一级审批通过后仍保持 `pending`
  - 第二级审批完成后才进入最终 `approved`
- 用户侧已新增访问申请历史列表。
- 数据集详情页已新增当前数据集的审批轨迹展示。
- 过往驳回原因现在可在审批轨迹和个人中心历史列表中直接查看。
- 审批通过时已支持配置授权有效期和最大下载次数：
  - `0` 天表示长期有效
  - `0` 次表示不限下载次数
- 普通下载和分批下载任务已经共用同一套授权消耗逻辑。
- 授权过期和次数耗尽后，接口会阻止继续下载或生成分批下载任务。
- 权限矩阵第一版已拉开：
  - `public`：登录后确认协议即可下载
  - `internal`：已通过开发者认证的用户可直接访问；未认证用户仍可走审批
  - `restricted`：仍需显式审批通过
- 审批请求创建时已补充提醒链：
  - 申请人会收到“已提交申请”通知
  - 数据集负责人和管理员会收到待审核提醒
- 后台审批列表已支持 SLA 视角：
  - 显示 SLA 时限、剩余时间、超时状态
  - 支持按 `SLA 内 / 已超时` 筛选
- 用户侧已可查看：
  - 当前授权是否有效
  - 到期时间
  - 已用次数 / 剩余次数
  - 审批进度
  - SLA 剩余时间 / 超时状态

仍缺：

- 更细权限矩阵

建议补齐：

- 用户侧：
  - 查看更细审批节点
- 后台侧：
- 权限规则侧：
  - 后续扩展到项目组、组织级、白名单级
- 授权侧：
  - 更细粒度授权规则
